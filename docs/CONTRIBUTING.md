# Contributing to Raptor.fitt

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a branch: `git checkout -b feature/amazing-feature`
4. Make changes
5. Test thoroughly
6. Commit: `git commit -m 'Add amazing feature'`
7. Push: `git push origin feature/amazing-feature`
8. Open Pull Request

## Development Guidelines

### Code Style
- Use TypeScript for type safety
- Follow existing naming conventions
- Use Tailwind CSS for styling
- Keep components small and focused

### Commit Messages
```
feat: Add AI coach endpoint
fix: Resolve SI calculation bug
docs: Update API reference
style: Format dashboard components
refactor: Simplify workout logging
test: Add unit tests for SI engine
```

### File Structure
```
- Pages go in app/
- Components in components/
- Utilities in lib/
- API routes in app/api/
```

## What to Contribute

### High Priority
- Authentication implementation
- AI coach features
- Recovery tracking page
- Unit tests
- Performance optimizations

### Medium Priority
- Additional charts
- Exercise library expansion
- Social features
- Export functionality

### Low Priority
- Theme customization
- Internationalization
- Advanced filtering

## Pull Request Process

1. Update README if needed
2. Add/update tests
3. Ensure no TypeScript errors
4. Test on mobile and desktop
5. Update documentation
6. Request review

## Code Review Checklist

- [ ] Code follows project style
- [ ] No console.logs left
- [ ] TypeScript types defined
- [ ] Components are responsive
- [ ] API routes validated
- [ ] Error handling added
- [ ] Performance considered
- [ ] Accessibility maintained

## Questions?

Open an issue for:
